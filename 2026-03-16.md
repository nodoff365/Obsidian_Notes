## 1. Docker
>**Docker**는 애플리케이션과 실행 환경을 **컨테이너(Container)** 형태로 패키징하여 어디서든 동일하게 실행할 수 있게 하는 **컨테이너 기반 가상화 플랫폼**입니다.
  [Docker Download](https://www.docker.com/)

## 2. WSL
>**WSL(Windows Subsystem for Linux)**은 Windows에서 **Linux 환경을 직접 실행할 수 있게 하는 기능**입니다. 별도의 가상머신 없이 Linux 명령어와 프로그램을 사용할 수 있습니다.
### 2.1 Windows 11에서 WSL이 많이 사용되는 이유
WSL은 **Windows 환경에서 Linux 도구를 함께 사용해야 하는 개발·교육 환경**에서 유용합니다.  
Microsoft 역시 개발 생산성을 높이는 환경으로 WSL을 안내하며, **VS Code, Git, Docker 등과 함께 사용하는 개발 흐름**을 권장합니다.
### 2.2 Docker와의 관계
Windows 11에서 **Docker Desktop은 일반적으로 WSL 2 기반 엔진**을 사용합니다.  
따라서 Docker 실습을 진행하려면 **WSL 2 설치와 정상 동작이 먼저 확보되어야 합니다.**
### 2.3 장점
WSL의 주요 장점은 다음과 같습니다.
- 듀얼 부팅 없이 Windows와 Linux 동시 사용 가능
- 기존 가상머신보다 가볍고 실행 속도가 빠름
- Windows 도구와 Linux CLI를 함께 활용 가능
- Docker 및 개발 환경 구축이 비교적 간단
---
Windows 기능 켜기/끄기 > `Linux용 Windows 하위 시스템,  가상 머신 플랫폼` 체크

PowerShell > `wsl` > `wsl --update`
 
 ```shell
wsl --update # 업데이트
wsl --version # 버전 확인
wsl --list --online # 설치 가능한 리눅스 목록
wsl --list # 설치된 배포판 목록
wsl --install Ubuntu-24.04 # Ubuntu-24.04 설치
wsl --set-default Ubuntu # 이후 `wsl` 실행 시 Ubuntu 환경이 자동 실행
wsl -d Ubuntu-24.04 # 특정 배포판 실행
 ```

## 3. Dockerfile 실습 

### 3.1 실습 목표
간단한 Flask 웹앱을 만들고, 이를 Docker 이미지로 빌드한 뒤 컨테이너로 실행합니다.
Docker 문서에 따르면 Dockerfile은 이미지 빌드에 필요한 지시문을 담은 텍스트 파일이며, `FROM`, `WORKDIR`, `COPY`, `RUN`, `CMD` 같은 지시문을 사용합니다.
### 3.2 작업 폴더 생성
```shell
mkdir -p c:\work\dockerfile-flask
cd c:\work\dockerfile-flask
code .
```
### 3.3 app.py 작성
```python
from flask import Flask
  
app = Flask(__name__)
  
@app.route("/")
def hello():
    return "<h1>Hello Flask in Docker!</h1>"
  
if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5000)
```
### 3.4 requirements.txt 작성
```
flask
```
### 3.5 Dockerfile 작성
```shell
FROM python:3.12-slim
  
WORKDIR /app
  
COPY requirements.txt ./
RUN pip install --no-cache-dir -r requirements.txt
  
COPY app.py ./
  
EXPOSE 5000
  
CMD ["python", "app.py"]
```
### 3.6 이미지 빌드
```shell
docker build -t myflask:v1 .
```
### 3.7 컨테이너 실행
```shell
docker run -d --name myflask1 -p 8081:5000 myflask:v1
```
### 3.8 확인
```shell
docker ps
docker logs myflask1

# 브라우저 접속
http://localhost:8081
```

## 4. 볼륨(volume) 실습
### 4.1 실습 목표
컨테이너를 삭제해도 데이터가 유지되는지 확인합니다.
Docker 문서에 따르면 볼륨은 Docker가 관리하는 영구 저장소이며, 컨테이너를 제거해도 데이터가 남습니다. 반면 bind mount는 호스트(Windows 11) 디렉터리를 직접 연결하며, 기본적으로 호스트 파일시스템에 쓰기 권한을 가질 수 있어 주의가 필요합니다.
### 4.2 볼륨 생성
```shell
docker volume create mydata

docker volume ls
```
### 4.3 Ubuntu 컨테이너로 파일 저장
```shell
docker run --name voltest1 -it --mount source=mydata,target=/data ubuntu bash
```

```shell
# 컨테이너 안에서
echo "Docker Volume Test" > /data/test.txt
cat /data/test.txt
exit
```
### 4.4 컨테이너 삭제
```shell
docker rm -f voltest1

docker ps -a # 삭제 확인
```
### 4.5 새 컨테이너에서 같은 볼륨 확인
```shell
docker run --name voltest2 -it --mount source=mydata,target=/data ubuntu bash
```

```shell
# 컨테이너 안에서
cat /data/test.txt
exit
```
### 4.6 확인
`Docker Volume Test`가 그대로 보이면 성공입니다.
즉, 컨테이너는 사라졌지만 볼륨 데이터는 유지된 것입니다.
