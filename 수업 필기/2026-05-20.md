```text
1.취약점이 없는 시스템의 경우
	1.1 서비스 자체의 취약점 공격
		1.1.1 DHCP: Discover 메세지에 대한 응답으로 Offer 메세지 전송
			이때 Mac Address를 속여 무작위로 Discover 메세지를 전송
			결국 Offer 메세지에서 응답한 IP Address 자원은 한정적
			따라서 DHCP에서 서비스할 IP자원이 고갈됨
			DHCP Starvation 공격(DDOS Attack)
		1.1.2 DNS: 외부 순환쿼리시 사용자의요청에 의한 URL을 IP Address로 변경해서 전달
		이때 해당 Name Server에서의 응답을 검증하지 않음. 
		공격자가 지속적으로 Public DNS에서 가짜 응답을 전송함.
		Public DNS는 해당 응답을 Cache에 저장후 사용자에게 Attack가 전송한 IP Address를 전달(DNS Cache Poisoning Attack)
		1.1.3 SSH를 통한 터널링: 고객의 요청이 없는 한 요구사항 이외의 서비스는 임의로 구성하지 말아야 함. 사용하지 않는 원격 데스크탑 서비스를 구성 후 방화벽에서 차단함.
		SSH 터널링을 이용하면 방화벽에서 차단한 서비스에 접속가능
```
