## DNS 확인 절차
```text
사용자가 www.bank.kr 입력
│
├─ ① 브라우저/OS 메모리 캐시 확인
│      ├─ 있음 → IP 반환
│      └─ 없음 ↓
│
├─ ② /etc/hosts 파일 확인
│      ├─ 있음 → IP 반환
│      └─ 없음 ↓
│
├─ ③ Local DNS 캐시 확인
│      ├─ 있음 → IP 반환
│      └─ 없음 ↓
│
├─ ④ 1차 DNS 서버에 질의
│      ├─ 캐시 있음 → IP 반환
│      └─ 캐시 없음 → 재귀 조회 시작
│              ├─ Root DNS : "TLD 서버 주소 알려줌"
│              ├─ TLD DNS  : "권한 DNS 주소 알려줌"
│              └─ 권한 DNS : "실제 IP 반환"
│
└─ ⑤ 1차 DNS 응답 없음 → 2차 DNS 서버에 질의
           ├─ 캐시 있음 → IP 반환
           └─ 캐시 없음 → 재귀 조회 시작
                   ├─ Root DNS : "TLD 서버 주소 알려줌"
                   ├─ TLD DNS  : "권한 DNS 주소 알려줌"
                   └─ 권한 DNS : "실제 IP 반환"
```
핵심 원칙

- 앞 단계에서 IP를 찾으면 **이후 단계는 즉시 생략**
- 재귀 조회는 **1차/2차 DNS 서버가 직접 수행**, 사용자 PC는 결과만 받음
- 모든 단계 실패 시 **DNS 오류(도메인 찾을 수 없음)** 반환
