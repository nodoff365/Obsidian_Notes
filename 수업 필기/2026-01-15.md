### L1 리피터, 허브 
- 전기 신호 재생, 증폭
- 단일 세그먼트
### L2 브리지, 스위치
- 충돌 영역 분리
- 다수의 세그먼트
### L3 라우터
- 브로드캐스트 영역 분리 

| 장비  | 포트수 | 충돌영역 | 브로드캐스트영역 |
| :-: | :-: | :--: | :------: |
| 허브  | 24  |  1   |    1     |
| 스위치 | 24  |  24  |    1     |
| 라우터 | 24  |  24  |    24    |
- **허브(L1)**
    
    - 모든 포트가 신호를 공유 → 충돌 영역 1, 브로드캐스트 영역 1
    
- **스위치(L2)**
    
    - 포트 별 충돌 영역 분리
    - 브로드캐스트는 전체 전달 → 브로드캐스트 영역 1  
	 (VLAN 없다는 전제)

- **라우터(L3)**
    
    - 포트(인터페이스)마다 네트워크 분리
    - 브로드캐스트 차단 → 포트 수만큼 브로드캐스트 영역

# 네트워크 토폴로지
##### 네트워크의 구성 요소인 노드와 링크 연결 형태를 말함
---
- 물리적 토폴로지
		![[Pasted image 20260115154522.png]]
	1. 버스(Bus)
	2. 링(Ring)
	3. 성(Star)
	    - LAN에 적합
	4. 망(Mesh)
		- WAN에 적합
		- 링크 수 : n(n-1)/2, 인터페이스 수 : n-1 `n = 노드 수`
	5. 계층, 목(Tree)
	
- 논리적 토폴로지
	1. 버스(Bus)형 : 데이터가 모든 방향으로 전송되는 구조, 동시에 모든 노드로 전파, 경쟁(Contention) 기법
	
	2. 링(Ring)형 : 데이터가 한쪽 방향으로 전송, 하나의 노드에서 하나의 노드로 전파, 토큰 패싱(Token Passing) 기법

# 인터넷 주소 개요
___
##### IPv4
- 32bit(4byte) 길이로 구성된 논리적인 주소 체계

| 클래스 | 시작 비트 |             주소 범위             |  사용처   |
| :-: | :---: | :---------------------------: | :----: |
|  A  |   0   |  `0.0.0.0 ~ 127.255.255.255`  |  대규모   |
|  B  |  10   | `128.0.0.0 ~ 191.255.255.255` |  중·대형  |
|  C  |  110  | `192.0.0.0 ~ 223.255.255.255` |  소규모   |
|  D  | 1110  | `224.0.0.0 ~ 239.255.255.255` | 멀티캐스트  |
|  E  | 1111  | `240.0.0.0 ~ 255.255.255.255` | 예비, 연구 |

- 사설 IP 주소 범위 (RFC 1918)

| 클래스 |                주소 범위                |
| :-: | :---------------------------------: |
|  A  |     `10.0.0.0 ~ 10.255.255.255`     |
|  B  |    `172.16.0.0 ~ 172.31.255.255`    |
|  C  | `192.168.0.0 ~ 192.168.255.255`<br> |
