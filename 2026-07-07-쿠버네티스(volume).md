### 1. Pod 정의 (nginx.yml)
`volumes`에서 `sgm-vol`이라는 emptyDir 볼륨을 만들고 → `volumeMounts`에서 그 볼륨을 컨테이너의 `/test1`에 마운트
```yaml
# empty dir 라이프 사이클 똑같음, pod 삭제 시 볼륨도 삭제
vi nginx.yml

apiVersion: v1
kind: Pod
metadata:
  name: nginx
  labels:
    app: nginx
    env: devel
    stor: ssd
spec:
  containers:
  - name: n1
    image: nginx
    imagePullPolicy: IfNotPresent
    ports:
    - containerPort: 80
    volumeMounts:
    - mountPath: /test1
      name: sgm-vol
  volumes:
  - name: sgm-vol
    emptyDir: {}
```

### 2. 생성 및 배치 확인
```bash
kubectl apply -f nginx.yml
kubectl get pod -o wide # Pod가 배치된 node 확인
```

### 3. node에서 실제 볼륨 위치 찾기
- Pod가 배치된 node로 이동한 뒤 실행
- emptyDir 볼륨은 node의 `/var/lib/kubelet/pods/<pod-uid>/volumes/kubernetes.io~empty-dir/<볼륨명>` 경로에 실제로 존재
```bash
find / -name sgm-vol
# 결과 예시:
# /var/lib/kubelet/pods/<pod-uid>/volumes/kubernetes.io~empty-dir/sgm-vol
```

### 4. node에서 파일 생성
```bash
cat > /var/lib/kubelet/pods/<pod-uid>/volumes/kubernetes.io~empty-dir/sgm-vol/sgm.txt <<eof
> emptyDir test
> eof
```

### 5. 컨테이너 안에서 확인
- node에 만든 파일이 컨테이너의 마운트 경로(`/test1`)에 그대로 보임
```bash
kubectl exec nginx -- ls /test1
kubectl exec nginx -- cat /test1/sgm.txt
# 출력: emptyDir test
```

### 6. Pod 삭제 시 볼륨도 삭제되는지 확인
- Pod가 삭제되면 `/var/lib/kubelet/pods/<pod-uid>/` 디렉터리 전체가 제거되므로, 그 안에 있던 emptyDir 볼륨(sgm.txt 포함)도 함께 사라짐. 이것이 emptyDir가 임시 저장소인 이유
```bash
# 삭제 전: node에서 볼륨 경로가 존재함
find / -name sgm-vol
# /var/lib/kubelet/pods/<pod-uid>/volumes/kubernetes.io~empty-dir/sgm-vol

# master에서 Pod 삭제
kubectl delete pod nginx

# 삭제 후: node에서 다시 찾으면 사라짐
find / -name sgm-vol
# (결과 없음)
```

## 2. node 라벨(label) 지정 정리
- Pod를 특정 node에 배치하는 방법을 실습한다. 방법은 nodeName(이름 직접 지정)과 nodeSelector(라벨 기준 선택) 두 가지이다.

### 2.1 nodeName 방식 (apache.yml)
- node 이름을 직접 지정한다.
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: apache
  labels:
    app: apache
    env: devel
spec:
  containers:
  - name: h1
    image: httpd
    imagePullPolicy: IfNotPresent
    ports:
    - containerPort: 80
  nodeName: node1
```
- `nodeName: node1` → 스케줄러를 거치지 않고 **node1**에 직접 배치된다.

### 2.2 node에 라벨 지정
- 다음 nodeSelector 방식을 쓰려면 먼저 node에 라벨이 있어야 한다.
- node에 라벨을 붙이면, Pod를 특정 node에 배치할 때 활용할 수 있다. 여기서는 각 node에 `stor`(스토리지 종류) 라벨을 지정한다.
```bash
kubectl label nodes node1 stor=hdd    # node1 → hdd
kubectl label nodes node2 stor=ssd    # node2 → ssd
kubectl label nodes node3 stor=nvme   # node3 → nvme

kubectl get nodes --show-labels # 확인
```

### 2.2.1 nodeSelector 방식 (alpine.yml)
- 라벨을 기준으로 node를 선택한다.
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: alpine
  labels:
    env: devel
spec:
  nodeSelector:
    stor: nvme
  containers:
  - name: a1
    image: alpine
    imagePullPolicy: IfNotPresent
    command: [ "tail", "-F", "/home" ]
```
- `nodeSelector: stor: nvme` → `stor=nvme` 라벨이 붙은 **node3**에 배치된다.  
- `command: [ "tail", "-F", "/home" ]`는 alpine이 바로 종료되지 않도록 실행 상태를 유지시킨다.

### 2.2.2 확인
- apache는 node1, alpine은 node3에 배치되어야 정상
```bash
kubectl get pod -o wide   # 각 Pod가 어느 node에 배치됐는지 확인
```

## 3. hostPath 실습 정리

### 1. nginx Pod (nodeName 방식)

**nginx.yml**
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx
  labels:
    app: nginx
spec: 
  nodeName: node1
  containers:
  - name: n1
    image: nginx
    imagePullPolicy: Never
    ports:
    - containerPort: 80
    volumeMounts:
    - mountPath: /usr/share/nginx/html/
      name: sgm-vol
  volumes:
  - name: sgm-vol
    hostPath:
      path: /html
      type: DirectoryOrCreate
```
- `nodeName: node1` : node1에 직접 배치
- hostPath : node1의 `/html` → 컨테이너 `/usr/share/nginx/html/` 마운트
- `type: DirectoryOrCreate` : 디렉터리 없으면 생성

**실행**
```bash
kubectl apply -f nginx.yml
kubectl get pods -o wide      # IP 확인
curl <ip>                     # 이 시점엔 index.html 없어서 403 또는 기본페이지
```

**node1에서 웹 파일 생성**
```bash
vi /html/index.html
```

```html
<html>
<body>
<h1>HOSTPATH-TEST</h1>
</body>
</html>
```

**다시 확인**
```bash
curl <ip>                     # HOSTPATH-TEST 출력
```

### 2. httpd Pod (nodeSelector 방식)

**http.yml**
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: httpd
  labels:
    app: apache
spec:
  nodeSelector:
    stor: hdd
  containers:
  - name: h1
    image: httpd
    imagePullPolicy: Never
    ports:
    - containerPort: 80
    volumeMounts:
    - mountPath: /usr/local/apache2/htdocs
      name: sgm-vol1
    - mountPath: /usr/local/apache2/htdocs/index.html
      name: sgm-file1
  volumes:
  - name: sgm-vol1
    hostPath:
      path: /html
      type: Directory
  - name: sgm-file1
    hostPath:
      path: /html/index.html
      type: File
```
- `nodeSelector: stor: hdd` : `stor=hdd` 라벨이 붙은 노드에 배치
- 볼륨 2개 마운트
    - `sgm-vol1` : node의 `/html` → `/usr/local/apache2/htdocs` (디렉터리, `type: Directory`)
    - `sgm-file1` : node의 `/html/index.html` → 같은 경로의 `index.html` (파일, `type: File`)

**실행**
```bash
kubectl apply -f http.yml
kubectl get po -o wide        # IP 확인
curl <ip>                     # index.html 내용 출력
```
