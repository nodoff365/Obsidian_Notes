### 리피터
데이터 통신에서 신호의 감쇠를 복구하고 재전송하는 장치
___
리피터 3R
- **Re-amplifying (재증폭):** 약해진 신호의 세기를 증폭
- **Re-shaping (재성형):** 찌그러진 신호 파형을 원래의 깨끗한 형태로 복원
- **Re-timing (재타이밍):** 신호의 타이밍(클럭)을 재조정하여 지터(Jitter)를 제거하고 정확한 데이터 전송을 보장

# L2 데이터 링크 계층, Data Link
---
- 프레이밍, 물리 주소 지정, 오류 제어, 흐름 제어, 접속 제어

```

								Parity
							↗
				오류 검출
			↗               ↘  
L2 오류제어                      CRC/FCS
			↘
				오류 정정       FEC (Forward Error Correction)
							
							↘   ARQ       Stop-and-Wait
																				
									   ↘   Sliding Window      Go-Back-N 
															 ↘  Selective Repeat
																			     
```

- **오류 검출** : 프레임 오류 여부만 확인 
	1. **Parity** : 간단한 1비트 체크
	2. **CRC/FCS** : 다중 비트 오류 검출 가능
<br>
- **오류 정정** : 오류 발생 시 데이터 복구
	1. **FEC** : 수신자가 재전송 없이 오류를 스스로 정정
	2. **ARQ (Automatic Repeat reQuest)** : 오류 발생 시 재전송 기반
		1. **Stop-and-Wait** : 한 프레임 전송 후 ACK 대기
		2. **Sliding Window** : 여러 프레임 연속 전송 가능
			1. **Go-Back-N** : 오류 발생 프레임부터 모두 재전송
			2. **Selective Repeat** : 오류 난 프레임만 선택 재전송
