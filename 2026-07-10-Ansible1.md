## 1. Ansible
### 1.1 Ansible 개요
* Red Hat에서 관리하는 오픈소스 구성관리(Configuration Management) 도구
* Agentless 방식: 관리 대상 노드에 별도 에이전트 설치 없이 SSH로 통신
* Python 기반, YAML 문법으로 작업(Task)을 정의
* 선언적(Declarative) 방식: "어떻게" 보다 "어떤 상태여야 하는지"를 기술
* 멱등성(Idempotency): 같은 플레이북을 여러 번 실행해도 결과가 동일하게 유지됨
* 주요 구성 요소
	- Control Node: Ansible이 설치되어 명령을 내리는 서버
	- Managed Node: 관리 대상 서버 (별도 에이전트 불필요, SSH + Python만 있으면 됨)
	- Inventory: 관리 대상 서버 목록 (`/etc/ansible/hosts`)
	- Module: 실제 작업을 수행하는 단위 (ping, shell, copy, yum 등)
	- Playbook: 여러 Task를 순서대로 정의한 YAML 파일
	- Ad-hoc Command: 플레이북 없이 단발성으로 실행하는 명령 (`ansible -m`, `-a`)

| rocky9-4 | 관리 대상               | node3 |
### 2.1 호스트명 설정

> 여러 서버를 IP로만 다루면 헷갈리기 때문에, 역할이 드러나도록 이름을 붙여둔다. 이렇게 하면 이후 작업에서 어떤 서버가 제어 노드고 어떤 서버가 관리 대상인지 바로 구분된다.

```bash
# rocky9-1
hostnamectl set-hostname cont

# rocky9-2
hostnamectl set-hostname node1

# rocky9-3
hostnamectl set-hostname node2

# rocky9-4
hostnamectl set-hostname node3
```


### 2.2 Ansible 설치 (rocky9-1, 제어 노드)

> Ansible은 기본 저장소에 포함돼 있지 않기 때문에, EPEL 저장소를 먼저 추가해야 설치할 수 있다. 
> 설치가 끝나면 `--version`으로 버전과 config 파일 경로를 확인하는데, 이렇게 확인해두면 나중에 설정을 변경할 때 어느 파일을 고쳐야 하는지 헷갈리지 않는다.

```bash
dnf install -y epel-release
dnf install -y ansible

ansible --version
```

### 2.3 SSH 키 생성 및 배포

> Ansible은 각 노드를 SSH로 제어하는데, 매번 비밀번호를 입력해야 한다면 자동화가 성립하지 않는다. 그래서 키 기반 인증을 미리 설정해, 비밀번호 없이 접속되도록 만든다.

```bash
ssh-keygen -b 2048 -t rsa -m PEM -q -N ""

scp .ssh/id_rsa.pub root@10.0.0.11:/root/.ssh/authorized_keys
scp .ssh/id_rsa.pub root@10.0.0.12:/root/.ssh/authorized_keys
scp .ssh/id_rsa.pub root@10.0.0.13:/root/.ssh/authorized_keys
scp .ssh/id_rsa.pub root@10.0.0.14:/root/.ssh/authorized_keys
```


### 2.4 최초 SSH 접속 (host key 등록)

> SSH는 처음 접속하는 서버를 신뢰할 수 있는지 판단할 방법이 없기 때문에, 최초 접속 시 host key 등록(yes/no) 절차를 거친다. 그런데 Ansible이 여러 대에 동시 접속하면 이 프롬프트에 응답할 수 있는 건 한 번에 하나뿐이라 나머지는 실패한다. 그래서 미리 한 대씩 수동으로 접속해 `known_hosts`에 등록해두는 것이다.

```bash
ssh root@10.0.0.11
ssh root@10.0.0.12
ssh root@10.0.0.13
ssh root@10.0.0.14
```


### 2.5 인벤토리 파일 작성

> Ansible이 명령을 내리려면 먼저 어떤 서버들을 관리 대상으로 삼을지 알아야 한다. 이 목록을 적어두는 파일이 인벤토리이며, 그룹으로 나눠두면 이후 "web 그룹에만 명령 내리기"처럼 대상 범위를 좁혀서 실행할 수 있다.

```bash
vi /etc/ansible/hosts
```

```ini
[all]
10.0.0.[11:14]

[node]
10.0.0.12
10.0.0.13
10.0.0.14

[web]
10.0.0.12

[was]
10.0.0.13

[db]
10.0.0.14

[http:children]
web
was
```


### 2.6 연결 테스트 (ping)

> SSH 키 설정과 인벤토리 작성이 제대로 됐는지 확인해야 다음 단계로 넘어갈 수 있기 때문에, ping 모듈로 헬스체크를 한다. `pong` 응답이 오면 이 노드들에 명령을 내릴 준비가 끝났다는 뜻이다.

```bash
ansible -i /etc/ansible/hosts all -m ping
ansible node -m ping
```
