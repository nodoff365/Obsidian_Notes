### init
부팅 완료 후 가장 먼저 실행되는 핵심 프로세스(PID 1)로, 시스템 모든 프로세스의 부모이며, 시스템 서비스들을 초기화하고 관리하며, 사용자가 로그인할 수 있는 환경을 만드는 역할을 하는 데몬

| Runlevel                   | 특징                                                                                      |
| :------------------------- | :-------------------------------------------------------------------------------------- |
| 0 (Halt)                   | - 시스템 중지  <br>- 기본값으로 설정 불가                                                             |
| 1 (Single user mode)       | - 단일 사용자 모드  <br>- 로그인 과정없이 root계정으로 로그인  <br>- 네트워크, 서버, 파일 공유 등 같은 서비스 사용 안함          |
| 2 (Multiuser, without NFS) | - 네트워크를 사용하지 않는 다중 사용자 모드  <br>- 여러 계정으로 로그인 가능  <br>- Rullevel 3에서 네트워크를 사용하지 않는 것과 동일 |
| 3 (Full multiuser mode)    | - 네트워크를 지원하는 다중 사용자 모드                                                                  |
| 4 (Unused)                 | - 사용되지 않는 Runlevel (사용자 정의 시 사용 가능)                                                     |
| 5 (X11)                    | - X window를 사용하는 다중 사용자모드 (그래픽 인터페이스)                                                   |
| 6 (Reboot)                 | - 시스템을 재기동할 때 사용  <br>- 기본값으로 설정 불가                                                     |

### Shell

##### zsh
- shell의 한 종류 (bash, sh와 동급)
- macOS, Kali Linux 기본 쉘
- bash 호환성 높음 (대부분 명령 그대로 사용 가능)

- 강력한 자동완성 기능
- 명령어 추천 및 하이라이트 제공
- 플러그인/테마 지원 (oh-my-zsh)
- git 브랜치 상태, 변경사항 표시 가능
- 사용자 설정 자유도 높음 (`.zshrc`)
##### 장점
- 생산성 높음
- 긴 명령어, 복잡한 작업에 유리
- 시각적으로 직관적
##### 단점
- 설정 많아지면 복잡해질 수 있음
- 서버 환경은 bash가 기본인 경우 많음

### 연결 연산자
##### `;` (세미콜론)
`command1 ; command2`
- 순차 실행, 성공/실패와 상관없이 다음 명령 실행
- `command1` 끝나면 무조건 `command2` 실행

##### `&&` (AND)
`command1 && command2`
- 앞 명령 성공 시에만 다음 명령 실행
- `command1`이 0(성공) → `command2` 실행 
- `command1`이 실패 → `command2` 실행 안 됨

##### `|` (파이프)
`command1 | command2`
- 앞 명령 출력 → 뒤 명령 입력으로 연결
- `command1`의 표준 출력(stdout)을 `command2`의 표준 입력(stdin)으로 전달
- 명령 성공 여부와 상관없이 연결됨
