# 시스템 백업

### tar [ ]
- -c : 새 아카이브(묶음 파일)를 생성
- -t : 아카이브의 내용을 출력
- -x : 아카이브에서 파일을 추출(풀기)
- -v : 처리되는 파일 목록을 자세히 출력(Verbose)
- -z :  gzip 방식을 사용하여 압축 `#확장자 gz`
- -j  :  bzip2 방식을 사용하여 압축 `#확장자 bz2`
```bash
tar cvf [묶음파일명] [원본] #/etc를 etc.tar로 묶음
tar tvf [파일명] | more #내용 확인
tar xvf [파일명] #묶음 해제
tar xvzf

gzip [파일명] #압축
gunzip [파일명] #압축 해제
```

### tar / cp 차이점

|  tar   |       cp       |
| :----: | :------------: |
| 링크도 복사 | 링크된 파일의 내용이 복사 |

tar cvf - system-release | (cd /home/nova/work/backup/;tar xvf -)
링크
```bash
ln [옵션] [원본] [링크] #링크파일 생성
```

1. 전체 경로 없이 풀기
```bash
tar -xvf archive.tar file.txt
```
- `archive.tar` 안에 `dir1/file.txt`가 들어있더라도  
    현재 디렉터리에 **`file.txt`만 추출**됨
- 경로 구조 무시 가능하지만, 하위 폴더에 있는 파일이면 충돌 위험 있음

2. 전체 경로 포함해서 풀기
```bash
tar -xvf archive.tar dir1/file.txt
```
- `archive.tar` 안의 경로 구조 그대로 재현
    - 현재 디렉터리에 `dir1/` 폴더 생성
    - 그 안에 `file.txt` 저장
- 원본 디렉터리 구조 유지

1. 전체 경로 포함해서 묶기
```bash
tar -cvf [파일명] [절대경로] #파일 경로 전체가 생성되기 때문에 지양
```
