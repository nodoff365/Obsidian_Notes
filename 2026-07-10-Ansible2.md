## Ansible 실습 (Ad-hoc / Playbook)

### 1. 개요

> 지난 글에서 환경 구성을 마쳤다. 이번 글에서는 Ansible이 명령을 실행하는 두 가지 방식을 다룬다. Ad-hoc은 한 줄짜리 명령을 즉석에서 실행하는 방식이고, Playbook은 여러 작업을 순서대로 정의해 파일로 저장, 재사용하는 방식이다.

---

### 2. Ad-hoc 명령 실습

> 플레이북 파일을 작성하지 않고도 한 줄짜리 명령을 여러 서버에 동시에 실행할 수 있는지 먼저 확인하기 위한 단계다. 간단한 일회성 작업이라면 파일 없이 이렇게 바로 처리할 수 있다.

`ansible node -m shell -a "mkdir /test"`
![](/assets/images/Ansible/2_1.png)
- `ansible node -m shell -a "mkdir /test"` 실행 결과, 10.0.0.12 / 10.0.0.13 / 10.0.0.14 세 노드 모두 `CHANGED | rc=0`으로 출력된다. `rc=0`은 명령이 정상 종료됐다는 뜻이라, `/test` 디렉토리가 세 노드 전부에 생성됐음을 확인할 수 있다.

`ansible node -m shell -a "ls -al / | grep test"`
![](/assets/images/Ansible/2_2.png)
- 세 노드 모두 `CHANGED | rc=0`으로 출력되며, `drwxr-xr-x root root ... test` 형태로 `test` 디렉토리 정보가 조회된다. `grep test`로 필터링했을 때 결과가 나온다는 건 `/test`가 실제로 존재한다는 뜻이다.

`ansible node -m shell -a "rm -rf /test"`
![](/assets/images/Ansible/2_3.png)
- 세 노드 모두 `CHANGED | rc=0`으로 출력된다. `rc=0`이므로 `/test` 삭제 명령이 정상적으로 처리됐음을 의미한다.

`ansible node -m shell -a "ls -al / | grep test"`
![](/assets/images/Ansible/2_4.png)
- 세 노드 모두 `FAILED | rc=1`, `non-zero return code`로 출력된다. 이건 에러가 아니라, `test` 디렉토리가 없어서 grep이 매칭할 대상을 찾지 못했다는 뜻이다 (grep은 결과가 없으면 종료코드 1을 반환한다). 즉 삭제가 정상적으로 된 것을 보여준다.

### 3. 플레이북 실습

> Ad-hoc 명령은 한 번 실행하면 사라지기 때문에, 여러 태스크를 순서대로 반복 실행하려면 YAML 파일로 저장해두는 편이 낫다. 그래서 플레이북 형태로 넘어간다.

#### 3.1 ls.yml
```bash
vi ls.yml
```

```yaml
---
- name: ls 디렉토리 확인
  hosts: node
  gather_facts: false
  ignore_errors: true

  tasks:
  - name: view / directory
    shell: ls -al /
```

```bash
ansible-playbook -i /etc/ansible/hosts ls.yml
```

![](/assets/images/Ansible/2_5.png)
- PLAY, TASK, PLAY RECAP 순서로 결과가 출력된다.
- TASK 단계에서 `changed: [10.0.0.13]`, `changed: [10.0.0.14]`, `changed: [10.0.0.12]`처럼 노드별로 실행 결과가 표시된다.
- PLAY RECAP에서 3개 노드 모두 `ok=1 changed=1 failed=0`으로 나오면 정상 실행된 것이다.

```bash
ansible-playbook -i /etc/ansible/hosts ls.yml -vv   # 상세 확인
```


#### 3.2 md.yml (생성 → 조회 → 삭제)

```bash
vi md.yml
```

```yaml
---
- name: make directory
  hosts: node
  gather_facts: false
  ignore_errors: true

  tasks:
  - name: make directory /test
    shell: mkdir /test

  - name: view directory
    shell: ls -al / | grep test

  - name: remove directory /test
    shell: rm -rf /test
```

```bash
ansible-playbook md.yml
```

![](/assets/images/Ansible/2_6.png)
- PLAY 안에 정의한 3개 TASK(`make directory /test`, `view directory`, `remove directory /test`)가 순서대로 실행되고, 각 TASK마다 대상 노드들의 결과가 출력된다.
- PLAY RECAP에서 3개 노드 모두 `ok=3 changed=3 failed=0 ignored=0`으로 나오면, 세 태스크 모두 에러 없이 정상적으로 실행된 것이다.
